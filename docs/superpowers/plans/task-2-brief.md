# Task 2: Alembic Setup and Initial Migration

## Task Description

Set up Alembic for database migrations and create the initial migration for the portfolio tables.

## Files to Create

1. `alembic/alembic.ini`
2. `alembic/env.py`
3. `alembic/versions/001_initial.py` (generated by Alembic)
4. `alembic/script.py.mako` (generated by Alembic)

## Files to Modify

1. `pyproject.toml` - add dependencies

## Requirements

1. Add dependencies to `pyproject.toml`:
   - `alembic>=1.13.0`
   - `asyncpg>=0.29.0`
   - `aiosqlite>=0.19.0`

2. Create `alembic/alembic.ini` with:
   - `script_location = alembic`
   - `sqlalchemy.url = postgresql+asyncpg://alpharadar:alpharadar@localhost:5432/alpharadar`

3. Create `alembic/env.py` that:
   - Imports `Base` from `alpharadar.infrastructure.portfolio.postgresql`
   - Sets `target_metadata = Base.metadata`
   - Configures async migrations using `async_engine_from_config`

4. Generate initial migration using:
   ```bash
   cd alembic && alembic revision --autogenerate -m "initial"
   ```

5. Verify migration works by running:
   ```bash
   alembic upgrade head
   ```

## Interfaces

- Consumes: `Base` from Task 1 (PostgreSQL models)
- Produces: Migration scripts, database schema

## Global Constraints

- Python 3.11+
- SQLAlchemy 2.0+ (async)
- Alembic 1.13+

## Success Criteria

- [ ] Dependencies added to pyproject.toml
- [ ] Alembic configuration files created
- [ ] Initial migration generated
- [ ] Migration can be applied (upgrade head)
